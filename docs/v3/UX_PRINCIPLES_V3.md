# Petit Stay — UX Principles V3

> 기준일: 2026-03-12
> "기능이 많은 베이비시터 앱"이 아니라
> "외국인 부모가 3분 안에 검증된 시터를 찾고 예약할 수 있는 다국어 매칭 플랫폼"

---

## 1. 소비자형 (Consumer-grade)

Petit Stay는 B2B SaaS가 아니다. 호텔 관리 시스템도 아니다.
**여행 중인 부모가 스마트폰으로 3분 안에 시터를 찾고 예약하는 소비자 서비스**다.

- 여행자는 앱을 설치하지 않는다. 웹 퍼스트 (PWA).
- 설명을 읽지 않아도 다음 단계가 보여야 한다.
- 한 손으로 조작할 수 있어야 한다.
- Creatrip, Klook, Airbnb처럼 "여행 중에 쓰는 서비스"의 기대치에 맞춰야 한다.

---

## 2. 모바일 퍼스트

- 디자인 기준은 **375px 폭** (iPhone SE). 여기서 완벽해야 한다.
- 데스크톱은 모바일 레이아웃을 넓힌 것이지, 별도 설계가 아니다.
- 터치 타깃 최소 44×44px.
- 스크롤은 세로만. 가로 스크롤, 가로 탭 바 최소화.
- 결제 플로우에서 키보드가 뷰를 가리지 않아야 한다.

---

## 3. 한 화면 한 작업 (One Screen, One Job)

| 원칙 | 예시 |
|---|---|
| 화면에는 하나의 주된 행동만 | 검색 결과 화면: "시터 선택"이 유일한 CTA |
| 보조 정보는 접어두거나 다음 화면에 | 시터 프로필: 핵심(사진/이름/평점/시급)은 상단, 상세(경력/자격/리뷰)는 스크롤 |
| 입력 폼은 한 번에 3~5개 필드 이하 | 예약 폼: 날짜/시간 → 아동 정보 → 확인/결제 (멀티스텝) |
| 완료 상태가 명확하게 보여야 | 결제 완료: "예약 확정" + 시터 정보 + 다음 단계 |

---

## 4. 신뢰 판단 속도 > 정보량

부모가 시터를 신뢰할지 말지는 **3초 안에** 결정된다. 정보를 많이 주는 것보다, 빠르게 판단할 수 있는 정보를 주는 것이 낫다.

### 시터 카드 (검색 결과)에서 보여야 하는 것

1. **얼굴 사진** — 진짜 사람이라는 첫 번째 신호
2. **이름 + 한 줄 소개** — 성격/스타일 직감
3. **별점 + 리뷰 수** — 사회적 증거
4. **핵심 배지 2~3개** — 신원확인✓, 언어등급, CPR
5. **시급** — 즉시 비교 가능
6. **가용 시간대** — 내 일정에 맞는지 즉시 확인

### 시터 프로필에서 신뢰 순서

1. 영상 소개 (30~60초) — 가장 강력한 신뢰 장치
2. 사진 + 이름 + 배지
3. 리뷰 (최근 3개 우선 표시)
4. 경력/자격 상세
5. 서비스 범위/가격/일정

## 5. 디자인 금지 사항

| 금지 | 이유 |
|---|---|
| **AI 템플릿/테마 디자인** | Wix/Squarespace 느낌. 소비자 서비스의 신뢰를 깎는다 |
| **B2B 대시보드 스타일** | 테이블, 사이드바, 복잡한 필터. 여행자는 이걸 기대하지 않는다 |
| **Lucide 아이콘 박스 나열** | 기능 설명을 아이콘 + 텍스트 그리드로 나열하는 패턴. 읽히지 않는다 |
| **스톡 사진 남발** | 가짜 가족, 가짜 시터 사진. 신뢰를 파괴한다 |
| **과도한 애니메이션** | 로딩 중이 아닌데 움직이는 것. 여행자는 기다리지 않는다 |
| **어두운 패턴 (dark pattern)** | 숨겨진 수수료, 해지 어렵게 만들기, 강제 업셀 |
| **정보 과잉 랜딩** | 스크롤이 10회 이상 필요한 랜딩. 검색 CTA까지 2스크롤 이내 |
| **작은 글씨로 면책** | 안전/보험/제외업무는 숨기지 않고 명확하게 |

## 7. 다국어 UX 원칙

- 언어 전환은 **항상 접근 가능** (헤더 또는 푸터)
- 언어 전환 시 현재 페이지 유지 (URL의 locale만 변경)
- CTA 버튼 텍스트는 각 언어에서 자연스러워야 한다 (번역투 금지)
- 날짜/시간 포맷은 locale에 맞춤 (미국: MM/DD, 일본: YYYY/MM/DD)
- 통화 표시는 **KRW 기준** + 참고 환산 (USD/JPY/TWD)
- RTL 언어는 초기 범위 밖

---

## 8. 접근성 기본

- 명암비 WCAG 2.1 AA 이상 (4.5:1 본문, 3:1 큰 텍스트)
- 포커스 상태 시각적 표시
- 이미지에 alt 텍스트
- 폼 요소에 label 연결
- 키보드 네비게이션 가능

---

## 9. 성능 기준

- LCP (Largest Contentful Paint) < 2.5s (모바일)
- FID (First Input Delay) < 100ms
- CLS (Cumulative Layout Shift) < 0.1
- 시터 검색 결과 로딩 < 1.5s
- 결제 페이지 로딩 < 2s
