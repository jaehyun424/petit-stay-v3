## 1. 제품 한 줄 정의

Petit Stay는 한국을 여행하는 외국인 가족이 숙소 유형에 관계없이 검증된 다국어 시터를 직접 검색·선택·예약·결제할 수 있도록 돕는 베이비시팅 매칭 플랫폼이다.

---

## 2. 사용자 4축

| 축 | 역할 | 핵심 니즈 |
|---|---|---|
| **부모 (Parent)** | 서비스 소비자. 시터를 직접 검색·선택·예약·결제·리뷰 | 낯선 나라에서 검증된 시터를 빠르고 안심되게 예약 |
| **시터 (Sitter)** | 독립 서비스 제공자. 프로필·가격·가용시간을 자율 등록하고 예약을 수락/거절 | 프리미엄 외국인 고객, 높은 시급, 유연한 일정, 커리어 자산 축적 |
| **파트너 (Partner)** | 유입 채널(referral/distribution partner). 호텔·서비스드 레지던스·에어비앤비 호스트·OTA·콘텐츠 파트너 | 투숙객 편의 향상, 브랜드 리스크 없이 돌봄 요청 처리, 부가수익 연결 |
| **운영자 (Admin)** | 플랫폼 운영. 시터 검증·배정 관제·분쟁 대응·정산·품질 관리 | 무사고 운영, 매칭 성공률·SLA 유지, 법률/보험 컴플라이언스 |

## 5. 핵심 퍼널

```
검색(Search) → 프로필(Profile) → 예약(Booking) → 결제(Payment) → 세션(Session) → 리뷰(Review)
```

| 단계 | 설명 |
|---|---|
| **검색** | 부모가 날짜·시간·아이 나이·위치·언어를 입력, 조건에 맞는 시터 목록 표시 |
| **프로필** | 시터 사진, 영상 소개, 자격/경력, 언어 등급, 리뷰 평점, 시급, 배지, 서비스 범위 |
| **예약** | 아동 정보(알레르기, 수면 루틴, 선호 활동), 비상 연락처, 서비스 범위·제외업무 동의 |
| **결제** | 해외카드(Visa/Master/JCB/UnionPay/AMEX) + Alipay/WeChat Pay. 안전결제(에스크로 또는 보호형 결제) |
| **세션** | 시터 도착 → 디지털 체크인 → 돌봄 수행 → 종료 리포트(활동 내역, 건강 상태, 특이사항) 자동 발송 |
| **리뷰** | 완료·결제된 세션 후에만 부모 리뷰 공개. 시터→부모 내부 평가 분리 |

## 6. Trust Stack (5층 신뢰 설계)

| 층 | 이름 | 내용 |
|---|---|---|
| 1층 | **검증 프로필** | 신원확인, 경력, 자격/교육, 레퍼런스, 언어 수준, 서비스 가능 연령대. Certified 배지 제도 |
| 2층 | **동영상 소개** | 시터 30~60초 자기소개 영상. 부모가 시터의 분위기와 언어 수준을 사전에 판단 |
| 3층 | **Verified Review** | 완료·결제된 세션 후에만 리뷰 공개. 운영 신뢰 지표(응답률, 취소율, 반복예약률, 마지막 활동일) |
| 4층 | **안전결제·분쟁대응** | 에스크로 또는 보호형 결제 구조, 환불/보류 규정, 이의제기 창구 |
| 5층 | **세션 리포트** | 체크인/종료 시각, 돌봄 요약, 이슈 로그. 부모·파트너 안심용 종료 리포트 |

### 신뢰 배지 목록

- 언어 등급 (L1/L2/L3)
- CPR/응급처치 자격
- 신원확인 완료
- 영상소개 등록
- 레퍼런스 확인 완료
- Certified 배지 (종합 검증 완료)

### 부정행위 방지

- 미이행·취소·직거래 유도 세션은 리뷰 불가
- 분쟁 중 예약은 검토 후 공개
- 가짜 리뷰 방지를 위한 결제 완료 조건

## 7. 파트너 역할

> 호텔·숙소는 **referral / distribution partner**이며, 고용주가 아니다.

| 파트너가 하는 일 | 파트너가 하지 않는 일 |
|---|---|
| QR코드·링크·가이드북으로 플랫폼을 투숙객에게 안내 | 시터를 고용하지 않음 |
| 투숙객 편의 향상, 부가서비스 연결 | 시터를 배정하지 않음 |
| 예약 현황 조회, 종료 리포트 열람 (콘솔 제공 시) | 시터 파견을 수요하지 않음 |

### 채널 우선순위

1. **SEO / 직접유입**: "babysitter in Seoul", 영어/일본어/중문 키워드
2. **외국인 관광 플랫폼 / OTA 제휴**: Creatrip, KKday 등
3. **숙소 파트너**: 호텔, 서비스드 레지던스, 에어비앤비 호스트, 디지털 가이드북
4. **커뮤니티**: 주한 외국인 부모 그룹, 여행 포럼, 추천 프로그램

---

## 8. 시터 역할

> 시터는 **독립 서비스 제공자**이며, 플랫폼의 직접고용 대상이 아니다.

| 시터가 하는 일 | 시터가 하지 않는 일 |
|---|---|
| 프로필 등록, 가격 자율 설정 | 플랫폼이 정한 고정 시급을 받지 않음 |
| 예약 수락/거절 자율 결정 | 강제 배정을 받지 않음 |
| 복수 플랫폼 활동 가능 (전속 아님) | 유니폼 의무 착용 아님 (디지털 ID + 사진 배지만 필수) |
| 세션 수행 및 종료 리포트 제출 | 투약, 의료행위, 목욕, 차량 이동 수행 않음 |

### 시터 언어 등급

| 등급 | 설명 | 초기 운영 기준 |
|---|---|---|
| L1 Safety | 기본 인계, 식사/화장실/잠/통증/위험 표현 소통 가능 | 공개 배정 비추천 |
| L2 Service | 부모 요청 이해, 문자 보고, 종료 리포트 가능 | 초기 기본 최소치 |
| L3 Premium | 자연스러운 응대, 돌발 상황 설명, 숙소 파트너와 소통 가능 | 호텔/프리미엄 채널 우선 |

### 시터 가격 가이드라인 (권장 범위, 강제 아님)

| 구분 | 권장 시급 | 대상 시터 |
|---|---|---|
| 일반 (에어비앤비·민박) | 15,000~25,000원 | 기본 외국어 소통 가능 |
| 스탠다드 (3~4성급 호텔) | 25,000~35,000원 | 외국어 유창 + 돌봄 경력 1년+ |
| 프리미엄 (5성급 호텔) | 35,000~50,000원 | 바이링궐 + 돌봄 경력 3년+ + Certified |

## 9. 언어 전략

| 단계 | 언어 | 용도 |
|---|---|---|
| 1차 런칭 | **EN + JP** | 부모 UX, 시터 프로필, 예약·결제·리뷰, 동의서, CS |
| 1.5차 | **ZH-TW** (전통중문) | 대만·홍콩 가족 시장 커버 |
| 2차 | **ZH-CN** (간체중문) | 중국 시장, CS·결제·동의서 운영 난도 고려 |
| 내부/심사 | **KO** | 운영 관리, 시터 온보딩, 법률/보험 문서, 정부 제출 |

### 언어 우선순위 근거

- 일본: 방한 365만(2024), 가족동반 비율 약 30%, 개별관광 96%
- 미국: 방한 148만, 최고 1인당 소비, 장기 체류, 프리미엄 호텔 선호
- 대만: 방한 189만, 가족여행 수요 증가 추세
- 홍콩: 방한 62만, 25% 가족, 영어 가능, 고소비
- 중국: 방한 548만, 가족 세그먼트 소비 평균 2배 → 2차 확장

## 10. 초기 KPI

| KPI | 목표 | 의미 |
|---|---|---|
| 매칭 성공률 | 90%+ (운영 SLA 95%) | 호텔 추천 채널 신뢰 유지의 최소선 |
| 리뷰 평점 | 4.5+ | 부모 만족도의 핵심 지표 |
| 중대 사고 | 0건 | 사업 지속 가능성의 절대 조건 |
| 월 거래건수 | 성장 추적 | PMF 신호 |
| 시터 재등록률 | 추적 | 공급 안정성 지표 |
| 정시 도착률 | 95%+ | 객실 내 서비스 특성상 가장 중요 |
| 종료 리포트 제출률 | 100% | 호텔/부모 신뢰와 분쟁 예방 |
| 반복 예약·소개 비중 | 30%+ | PMF 신호 |

## 12. 후속 기능과 비범위

### 후속 구현 (V2+)

- 호텔 전용 대시보드 고도화 (화이트라벨, 월 리포트)
- PMS 연동 (Mews Open API, Oracle OHIP)
- AI 자동매칭 / 추천 알고리즘
- 네이티브 앱
- 부산/제주/해외 확장
- MICE·웨딩·이벤트 케어
- 문화체험형 키즈 액티비티
- 내국인 호캉스/데이트나잇 보완 채널

### 비범위 (하지 않는 것)

- 시터 직접 고용
- 호텔 staffing 회사 포지셔닝
- 플랫폼 고정 시급/등급제 강제
- 호텔이 예약을 만들고 시터를 배정하는 운영형 플로우
- 실시간 영상 스트리밍
- 자동 동적 가격엔진 (초기)

## 13. 기술 스택

### V3 확정 스택

| 항목 | 선택 | 역할 |
